## The Fix

The "Continue" call-to-action on every journey card is a dead control. It renders a bare
`<Button>` with no `onClick` and no surrounding `<Link>`, and the `Button` primitive is a plain
`<button>` with no anchor/`asChild` support — so there is literally nothing for a click to do.
A learner who creates a journey and clicks "Continue" to resume goes nowhere. The fix points that
control at the journey's overview surface using the same `Link`-styled-as-button pattern the
dashboard's empty-state CTA already uses, so it becomes a real, preload-on-intent navigation with
no new primitive and no new route.

The one load-bearing decision is *where* "Continue" should land. The user asked for the "journey
overview," but there is **no `/journey/$journeyId` index route** — that path is a layout-only route
(`$journeyId.tsx`, renders an `<Outlet/>`) whose only children are `interview`, `progress`, and the
`waypoint/*` subtree. The journey's actual overview surface is the **progress/roadmap page**
(`/journey/$journeyId/progress` — "the learner's mastery map — waypoints, streaks, and review-due
concepts"). So "journey overview" resolves to `/journey/$journeyId/progress`; a bare
`/journey/$journeyId` would render an empty Outlet (fuzzy not-found). This is confirmed at the gate.

## Restated Request

Make the "Continue" button on a journey card navigate to that journey's overview when clicked,
instead of doing nothing — so a returning learner can resume the journey they created.

## Acceptance Criteria

1. Clicking "Continue" on a journey card navigates to `/journey/<that journey's id>/progress`
   (the journey overview/roadmap), with the correct `journeyId` param interpolated per card.
2. The control keeps its current appearance (secondary button, "Continue" label + right-arrow
   icon) and its accessible name (`Continue <journey title>`); it is a real link (keyboard-
   focusable, right-click/⌘-click opens in new tab, intent-preload works).
3. No regression to the surrounding card layout (footer row: relative timestamp on the left,
   Continue on the right) in light and dark themes.

## Plan / Steps

1. **JourneyCard.tsx** — replace the dead `<Button>` in the footer with a router `Link` styled as
   a secondary button, mirroring the empty-state CTA:
   ```tsx
   import { Link } from "@tanstack/react-router";
   // …
   <Link
     to="/journey/$journeyId/progress"
     params={{ journeyId: journey.id }}
     className="btn-base btn-secondary btn-sm inline-flex items-center gap-1.5"
     aria-label={`Continue ${journey.title}`}
   >
     Continue
     <ArrowRight size={14} aria-hidden="true" />
   </Link>
   ```
